# API Design Exercise: Task Management API

This project is a simple RESTful API for managing tasks. It was created for an API design, mocking, and documentation exercise.

The API supports:

- Creating tasks
- Reading all tasks
- Reading one task by ID
- Updating tasks
- Deleting tasks
- Filtering tasks by status or priority
- A basic browser UI for testing the API

Real task IDs are generated by MongoDB, so they look like 24-character ObjectIds such as `6876b4a9c96f2c5d12345671`.

## Tech Stack

- Node.js
- Express
- MongoDB
- Mongoose
- Postman mock collection
- OpenAPI documentation

## MongoDB Requirement

You need one of these MongoDB options:

1. Local MongoDB installed on your computer
2. A free MongoDB Atlas cluster

For the easiest classroom submission, MongoDB Atlas is usually best because it gives you a hosted database connection string.

What you need from MongoDB:

- A database connection URI
- A database name, such as `task_management_api`
- If using Atlas, a database user/password and an allowed IP address

## Setup

Install dependencies:

```bash
npm install
```

Create your environment file:

```bash
cp .env.example .env
```

Update `.env` with your MongoDB connection string:

```text
PORT=3000
MONGODB_URI=mongodb+srv://xbuchanz:<db_password>@xbuchanz-dev.rcutwd8.mongodb.net/task_management_api?appName=xbuchanz-dev
```

Replace `<db_password>` with the password for the `xbuchanz` database user. Keep the database name `task_management_api` before the `?appName` part so this project stores tasks in the correct database.

For local MongoDB, the URI would look like:

```text
MONGODB_URI=mongodb://127.0.0.1:27017/task_management_api
```

Do not commit your real `.env` file. It is intentionally ignored by Git.

Run the API:

```bash
npm run dev
```

Or:

```bash
npm start
```

The local API will run at:

```text
http://localhost:3000/api/v1
```

The basic test UI will run at:

```text
http://localhost:3000
```

Optional: insert sample data into MongoDB:

```bash
npm run seed
```

## Endpoints

| Method | Endpoint | Description |
| --- | --- | --- |
| `GET` | `/api/v1/tasks` | Get all tasks |
| `GET` | `/api/v1/tasks/:id` | Get one task |
| `POST` | `/api/v1/tasks` | Create a task |
| `PATCH` | `/api/v1/tasks/:id` | Update a task |
| `DELETE` | `/api/v1/tasks/:id` | Delete a task |

## Project Files

- `src/` contains the working Express and MongoDB API.
- `public/` contains a simple browser UI for testing the API.
- `outputs/task-api-documentation.md` contains the written API documentation.
- `outputs/task-api-postman-collection.json` can be imported into Postman.
- `outputs/openapi.yaml` contains OpenAPI documentation.
- `outputs/mock-tasks.json` contains sample task data.
- `outputs/submission-notes.md` contains class submission notes.
